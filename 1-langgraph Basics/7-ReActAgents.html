ReAct Agent Architecture#

Aim#

This is the intuition behind ReAct, a general agent architecture.

  1. act - let the model call specific tools

  2. observe - pass the tool output back to the model

  3. reason - let the model reason about the tool output to decide what to do next (e.g., call another tool or just respond directly)

## State Schema
from typing_extensions import TypedDict
from langchain_core.messages import AnyMessage
from typing import Annotated
from langgraph.graph.message import add_messages
class State(TypedDict):
    messages:Annotated[list[AnyMessage],add_messages]

### Entire Chatbot With LangGraph
from IPython.display import Image, display
from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import ToolNode
from langgraph.prebuilt import tools_condition

### Node definition
def tool_calling_llm(state:State):
    return {"messages":[llm_with_tools.invoke(state["messages"])]}

# Build graph
builder = StateGraph(State)
builder.add_node("tool_calling_llm", tool_calling_llm)
builder.add_node("tools", ToolNode(tools))

builder.add_edge(START, "tool_calling_llm")
builder.add_conditional_edges(
    "tool_calling_llm",
    # If the latest message (result) from assistant is a tool call -> tools_condition routes to tools
    # If the latest message (result) from assistant is a not a tool call -> tools_condition routes to END
    tools_condition,
)
builder.add_edge("tools","tool_calling_llm")


graph = builder.compile()

# View
display(Image(graph.get_graph().draw_mermaid_png()))

What is machine learning
================================== Ai Message ==================================

**Machine learning (ML)** is a sub‑field of artificial intelligence that focuses on building systems that can automatically improve their performance on a task through experience (i.e., data), rather than being explicitly programmed for every possible situation.

### Core Idea
- **Data + Algorithm → Model**: An ML algorithm consumes a dataset, discovers patterns or statistical regularities, and produces a *model* (e.g., a set of parameters).  
- **Prediction/Decision**: The model is then used to make predictions, classify new inputs, or take actions on data it has never seen before.  
- **Learning**: As more data become available, the model can be updated (re‑trained) to improve accuracy or adapt to changes.

### Main Types of Machine Learning
| Category | How it learns | Typical tasks | Example algorithms |
|----------|---------------|---------------|--------------------|
| **Supervised learning** | Learns from labeled examples (input‑output pairs). | Classification, regression, object detection. | Linear regression, decision trees, support‑vector machines, neural networks. |
| **Unsupervised learning** | Finds structure in unlabeled data. | Clustering, dimensionality reduction, anomaly detection. | K‑means, hierarchical clustering, PCA, autoencoders. |
| **Semi‑supervised learning** | Uses a small amount of labeled data plus a large amount of unlabeled data. | Improves performance when labeling is costly. | Graph‑based methods, self‑training, consistency regularization. |
| **Reinforcement learning** | Learns by interacting with an environment and receiving reward signals. | Game playing, robotics, recommendation systems. | Q‑learning, policy gradients, deep Q‑networks. |
| **Self‑supervised learning** | Generates its own supervisory signal from raw data (e.g., predicting missing parts). | Language modeling, vision pre‑training. | Masked language models (BERT), contrastive learning. |

### Typical Workflow
1. **Define the problem** (e.g., predict house prices).  
2. **Collect & preprocess data** (cleaning, feature engineering, normalization).  
3. **Choose a model/algorithm** (e.g., random forest).  
4. **Train** the model on a training set (optimize parameters to minimize a loss function).  
5. **Validate** using a separate validation set to tune hyper‑parameters and avoid over‑fitting.  
6. **Test** on an unseen test set to estimate real‑world performance.  
7. **Deploy** the model for inference; monitor and retrain as needed.

### Why It Matters
- **Automation of complex tasks** that are hard to hand‑code (e.g., speech recognition, image classification).  
- **Scalability**: Models can handle massive datasets and make predictions in milliseconds.  
- **Adaptability**: Systems can improve over time as more data are gathered.  

### Real‑World Applications
- **Vision**: Face detection, medical imaging diagnostics, autonomous‑vehicle perception.  
- **Language**: Spam filtering, machine translation, chatbots, sentiment analysis.  
- **Recommendation**: Product suggestions on e‑commerce sites, content playlists.  
- **Finance**: Fraud detection, algorithmic trading, credit scoring.  
- **Healthcare**: Predictive patient risk scores, drug discovery, personalized treatment plans.  

### Key Challenges
- **Data quality & bias**: Poor or biased data lead to unreliable or unfair models.  
- **Interpretability**: Complex models (e.g., deep neural networks) can be “black boxes.”  
- **Generalization**: Overfitting to training data reduces performance on new data.  
- **Computational resources**: Training large models can be expensive in time and energy.  

### Bottom Line
Machine learning provides a systematic way for computers to *learn* from data, turning raw observations into actionable predictions or decisions. It underpins many of the intelligent systems we interact with daily and continues to expand into new domains as algorithms, data availability, and computing power improve.

MemorySaver#

LangGraph can use a checkpointer to automatically save the graph state after each step.

This built-in persistence layer gives us memory, allowing LangGraph to pick up from the last state update.

One of the easiest checkpointers to use is the MemorySaver, an in-memory key-value store for Graph state.

All we need to do is simply compile the graph with a checkpointer, and our graph has memory!

from langchain_openai import ChatOpenAI
llm = ChatOpenAI(model="gpt-4o")
llm_with_tools = llm.bind_tools(tools)

### Entire Chatbot With LangGraph
from IPython.display import Image, display
from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import ToolNode
from langgraph.prebuilt import tools_condition

### Node definition
def tool_calling_llm(state:State):
    return {"messages":[llm_with_tools.invoke(state["messages"])]}

# Build graph
builder = StateGraph(State)
builder.add_node("tool_calling_llm", tool_calling_llm)
builder.add_node("tools", ToolNode(tools))

builder.add_edge(START, "tool_calling_llm")
builder.add_conditional_edges(
    "tool_calling_llm",
    # If the latest message (result) from assistant is a tool call -> tools_condition routes to tools
    # If the latest message (result) from assistant is a not a tool call -> tools_condition routes to END
    tools_condition,
)
builder.add_edge("tools","tool_calling_llm")

<langgraph.graph.state.StateGraph at 0x1c7df206fc0>

from langgraph.checkpoint.memory import MemorySaver
memory=MemorySaver()
graph_memory = builder.compile(checkpointer=memory)

# View
display(Image(graph_memory.get_graph().draw_mermaid_png()))
